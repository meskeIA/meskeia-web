🔄 Fundamentos de Teoría de Colas

Sistemas de espera y análisis de congestión

🎯 ¿Qué es la Teoría de Colas?

La Teoría de Colas es una rama de la investigación operativa que estudia los sistemas de espera. Analiza situaciones donde los "clientes" llegan a un sistema, esperan en cola si es necesario, reciben servicio y después se van.

Ejemplos Cotidianos:

  • Banco: Clientes esperan para ser atendidos por cajeros
  • Hospital: Pacientes en urgencias esperando atención médica
  • Supermercado: Compradores en cajas registradoras
  • Call Center: Llamadas esperando a operadores
  • Servidor web: Peticiones HTTP esperando procesamiento

🏗️ Componentes de un Sistema de Colas

👥 Población de Clientes

Fuente: De donde vienen los clientes

Tamaño: Finita o infinita

Comportamiento: Pacientes o impacientes

📈 Proceso de Llegada

Patrón: Determinista o aleatorio

Tasa (λ): Clientes por unidad de tiempo

Distribución: Poisson es la más común

📋 Cola

Capacidad: Limitada o ilimitada

Disciplina: FIFO, LIFO, prioridades

Comportamiento: Balking, reneging

⚙️ Sistema de Servicio

Servidores: Uno o múltiples

Tasa (μ): Clientes servidos por tiempo

Distribución: Exponencial es común

⚡ Sistema M/M/1 - El Modelo Básico

Características del M/M/1:
  • Llegadas siguen un proceso de Poisson con tasa λ
  • Tiempos de servicio son exponenciales con tasa μ
  • Un solo servidor
  • Cola de capacidad infinita
  • Disciplina FIFO (primero en llegar, primero en salir)

📊 Fórmulas Principales:

ρ = λ/μ (Factor de utilización)
Debe ser < 1 para estabilidad del sistema
L = λ/(μ-λ) (Número promedio en el sistema)
Incluye clientes en cola + siendo servidos
Lq = λ²/[μ(μ-λ)] (Número promedio en cola)
Solo clientes esperando ser servidos
W = 1/(μ-λ) (Tiempo promedio en sistema)
Tiempo total desde llegada hasta salida
Wq = λ/[μ(μ-λ)] (Tiempo promedio en cola)
Tiempo esperando antes del servicio
💡 Ley de Little: L = λW y Lq = λWq
Relaciona el número promedio de clientes con el tiempo promedio de permanencia

❓ Preguntas Frecuentes - Teoría de Colas

¿Qué significa que ρ debe ser menor que 1? +

El factor de utilización ρ = λ/μ representa la fracción de tiempo que el servidor está ocupado.

  • ρ < 1: Sistema estable, la cola no crece indefinidamente
  • ρ = 1: Sistema crítico, el servidor está al 100% de capacidad
  • ρ > 1: Sistema inestable, la cola crece sin límite

Ejemplo: Si λ=8 y μ=10, entonces ρ=0.8 (80% de utilización, sistema estable)

¿Cómo interpreto el tiempo de espera en cola? +

Wq es el tiempo promedio que un cliente espera en cola antes de ser atendido.

  • No incluye el tiempo de servicio
  • Se mide en las mismas unidades que λ y μ
  • Si λ=5 clientes/hora y Wq=0.2 horas, cada cliente espera 12 minutos promedio

Tiempo total = Wq (espera) + 1/μ (servicio)

¿Por qué usar distribuciones Poisson y exponencial? +

Estas distribuciones tienen propiedades matemáticas que simplifican los cálculos:

  • Poisson: Modela llegadas aleatorias independientes
  • Exponencial: Tiene la propiedad "sin memoria"
  • Markoviano: El futuro solo depende del estado presente

En la práctica: Son aproximaciones razonables para muchos sistemas reales

¿Qué hago si mi sistema tiene ρ > 1? +

Un sistema con ρ > 1 es inestable. Soluciones posibles:

  • Aumentar μ: Mejorar el proceso de servicio
  • Reducir λ: Controlar las llegadas (citas, reservas)
  • Agregar servidores: Cambiar a modelo M/M/c
  • Capacidad limitada: Usar modelo M/M/1/K

Ejemplo práctico: Un banco con colas muy largas puede abrir más ventanillas

¿Cuándo NO usar el modelo M/M/1? +

El modelo M/M/1 NO es apropiado cuando:

  • Múltiples servidores: Usar M/M/c
  • Llegadas por lotes: Usar modelos de lotes
  • Prioridades: Usar modelos con prioridades
  • Tiempos fijos: Usar M/D/1 o D/D/1
  • Capacidad limitada: Usar M/M/1/K
  • Población finita: Usar M/M/1/N/N

🚚 Fundamentos del Problema de Transporte

Optimización de costos de distribución

🎯 ¿Qué es el Problema de Transporte?

El Problema de Transporte es un tipo especial de programación lineal que busca encontrar la forma más económica de transportar un producto desde varios puntos de origen (fuentes) hasta varios puntos de destino.

Ejemplos Prácticos:

  • Distribución: Desde almacenes a tiendas
  • Producción: Desde fábricas a centros de distribución
  • Logística: Reparto de productos a clientes
  • Recursos: Asignación de trabajadores a proyectos
  • Energía: Plantas de energía a ciudades

🧮 Formulación Matemática

Datos del problema:

  • m fuentes con ofertas s₁, s₂, ..., sₘ
  • n destinos con demandas d₁, d₂, ..., dₙ
  • Costos unitarios cᵢⱼ de fuente i a destino j
  • Variables xᵢⱼ = cantidad a transportar de i a j

Función objetivo (minimizar):

Z = Σᵢ Σⱼ cᵢⱼ × xᵢⱼ

Restricciones:

  • Oferta: Σⱼ xᵢⱼ = sᵢ (para cada fuente i)
  • Demanda: Σᵢ xᵢⱼ = dⱼ (para cada destino j)
  • No negatividad: xᵢⱼ ≥ 0

⚖️ Tipos de Problemas de Transporte

✅ Problema Balanceado

Σ Oferta = Σ Demanda

  • Toda la oferta se consume
  • Toda la demanda se satisface
  • Solución siempre existe
  • Más fácil de resolver

⚠️ Problema No Balanceado

Σ Oferta ≠ Σ Demanda

  • Sobra oferta o demanda
  • Se agregan fuentes/destinos ficticios
  • Costos ficticios = 0
  • Se convierte en balanceado
💡 Conversión a problema balanceado:
  • Exceso de oferta: Agregar destino ficticio con demanda igual al exceso
  • Exceso de demanda: Agregar fuente ficticia con oferta igual al exceso

🔧 Métodos de Resolución

1️⃣ Método de la Esquina Noroeste

Procedimiento:

  1. Comenzar en la celda superior izquierda (esquina noroeste)
  2. Asignar el máximo posible: min(oferta, demanda)
  3. Eliminar fila o columna que se agote
  4. Moverse a la siguiente celda disponible
  5. Repetir hasta completar la tabla
Ventaja: Muy simple de aplicar
Desventaja: No considera los costos, solución no óptima

2️⃣ Método del Costo Mínimo

Procedimiento:

  1. Identificar la celda con menor costo disponible
  2. Asignar el máximo posible en esa celda
  3. Eliminar fila o columna que se agote
  4. Repetir con la siguiente celda de menor costo
Ventaja: Considera costos, mejor solución inicial
Desventaja: Aún no garantiza optimalidad

3️⃣ Método de Vogel (VAM)

Procedimiento:

  1. Calcular penalizaciones para cada fila y columna
  2. Penalización = diferencia entre dos costos menores
  3. Seleccionar fila/columna con mayor penalización
  4. Asignar en la celda de menor costo de esa fila/columna
  5. Recalcular penalizaciones y repetir
Ventaja: Generalmente da la mejor solución inicial
Desventaja: Más complejo de calcular manualmente

❓ Preguntas Frecuentes - Problema de Transporte

¿Qué hago si mi problema no está balanceado? +

Problema no balanceado significa que la oferta total ≠ demanda total.

Solución:

  • Si sobra oferta: Agregar un destino ficticio con demanda igual al exceso
  • Si sobra demanda: Agregar una fuente ficticia con oferta igual al exceso
  • Costos ficticios = 0 (no hay costo real)

Ejemplo: Oferta=500, Demanda=400 → Agregar destino ficticio con demanda=100

¿Cuál es el mejor método para empezar? +

Depende de tus objetivos:

  • Aprendiendo: Esquina Noroeste (más simple)
  • Practicando: Costo Mínimo (balance entre simplicidad y calidad)
  • Optimizando: Método de Vogel (mejor solución inicial)

En la práctica: Usa software especializado como simplex, pero entender los métodos heurísticos ayuda a validar resultados.

¿Cómo sé si mi solución es óptima? +

Los métodos heurísticos dan soluciones iniciales, no necesariamente óptimas.

Para verificar optimalidad:

  • Método MODI (test de optimalidad)
  • Stepping Stone (mejora iterativa)
  • Método Simplex (garantiza optimalidad)

En esta aplicación: Los métodos implementados son heurísticos para enseñanza, no garantizan optimalidad total.

¿Puedo tener rutas prohibidas? +

Sí, a veces ciertas rutas no son factibles por restricciones físicas, políticas o económicas.

Solución: Asignar un costo muy alto (M) a las rutas prohibidas.

Ejemplo:

  • Costo normal: 10€
  • Ruta prohibida: 9999€ (valor M)

El algoritmo evitará automáticamente estas rutas al buscar el costo mínimo.

¿Qué significan las unidades en el problema? +

Consistencia de unidades es clave:

  • Ofertas/Demandas: Toneladas, cajas, litros, etc.
  • Costos: €/tonelada, $/caja, €/litro, etc.

Ejemplo consistente:

  • Oferta: 100 toneladas
  • Demanda: 80 toneladas
  • Costo: 15 €/tonelada
  • Resultado: 1.200€ total

📅 Fundamentos de la Planificación CPM

Método de la Ruta Crítica para gestión de proyectos

🎯 ¿Qué es el Método CPM?

CPM (Critical Path Method) es una técnica de planificación de proyectos que identifica la ruta crítica - la secuencia más larga de actividades que determina la duración mínima del proyecto completo.

Aplicaciones Comunes:

  • Construcción: Planificación de edificios, carreteras
  • Software: Desarrollo de aplicaciones, sistemas
  • Eventos: Organización de conferencias, bodas
  • Producción: Lanzamiento de nuevos productos
  • Investigación: Proyectos científicos, estudios

🧩 Conceptos Fundamentales

📋 Actividad

Tarea específica del proyecto

  • Consume tiempo y recursos
  • Tiene un inicio y fin definidos
  • Ejemplo: "Diseñar planos"

⏱️ Duración

Tiempo necesario para completar una actividad

  • Se estima en días, semanas, meses
  • CPM asume duraciones determinísticas
  • Ejemplo: 5 días

🔗 Dependencia

Relación entre actividades

  • Predecesora: debe terminar antes
  • Sucesora: puede empezar después
  • Ejemplo: A debe terminar antes que B

🛤️ Ruta Crítica

Secuencia más larga de actividades

  • Determina duración del proyecto
  • Sin holgura (slack = 0)
  • Ejemplo: A → C → F

🧮 Cálculos del Método CPM

➡️ Forward Pass (Cálculo Hacia Adelante)

Calcula los tiempos más tempranos de inicio y fin.

Fórmulas:

  • IT (Inicio Temprano) = max(FT de predecesoras)
  • FT (Fin Temprano) = IT + Duración

Procedimiento:

  1. Actividades sin predecesoras: IT = 0
  2. Para cada actividad: IT = max(FT de sus predecesoras)
  3. Calcular FT = IT + Duración
  4. Continuar hasta la actividad final

⬅️ Backward Pass (Cálculo Hacia Atrás)

Calcula los tiempos más tardíos de inicio y fin.

Fórmulas:

  • FT Tardío = min(IT Tardío de sucesoras)
  • IT Tardío = FT Tardío - Duración

Procedimiento:

  1. Actividades finales: FT Tardío = FT Temprano
  2. Para cada actividad: FT Tardío = min(IT Tardío de sucesoras)
  3. Calcular IT Tardío = FT Tardío - Duración
  4. Continuar hasta las actividades iniciales

⏰ Cálculo de Holguras

La holgura (slack) indica cuánto se puede retrasar una actividad sin afectar el proyecto.

Fórmula:

  • Holgura = IT Tardío - IT Temprano
  • También: Holgura = FT Tardío - FT Temprano

Interpretación:

  • Holgura = 0: Actividad crítica (no se puede retrasar)
  • Holgura > 0: Actividad con flexibilidad de tiempo

🎯 Identificación de la Ruta Crítica

🔴 Ruta Crítica: Secuencia de actividades con holgura = 0 que va desde el inicio hasta el final del proyecto.

Propiedades de la Ruta Crítica:

  • Determina la duración mínima del proyecto
  • Actividades sin margen de tiempo (holgura = 0)
  • Cualquier retraso en estas actividades retrasa todo el proyecto
  • Puede haber múltiples rutas críticas en algunos proyectos
  • Cambia dinámicamente según avance del proyecto

💡 Gestión de la Ruta Crítica:

  • Monitoreo constante: Revisar progreso diariamente
  • Recursos prioritarios: Asignar los mejores recursos
  • Planes de contingencia: Preparar alternativas
  • Comunicación clara: Informar a todos los stakeholders
  • Flexibilidad: Redistribuir recursos de actividades no críticas

❓ Preguntas Frecuentes - Planificación CPM

¿Qué pasa si una actividad crítica se retrasa? +

Impacto directo: Todo el proyecto se retrasa el mismo tiempo.

Estrategias de mitigación:

  • Crashing: Agregar recursos para acelerar
  • Fast tracking: Hacer actividades en paralelo
  • Replanificación: Cambiar secuencia de actividades
  • Reducir alcance: Eliminar actividades no esenciales

Ejemplo: Si "Cimentación" (crítica) se retrasa 3 días, todo el proyecto se retrasa 3 días.

¿Puedo usar la holgura de actividades no críticas? +

¡Sí! Las actividades con holgura > 0 ofrecen flexibilidad.

Usos estratégicos de la holgura:

  • Redistribución de recursos: Mover personal a actividades críticas
  • Optimización de costos: Usar recursos más económicos
  • Gestión de riesgos: Buffer para imprevistos
  • Mejorar calidad: Tiempo extra para revisiones

Cuidado: No uses toda la holgura - deja margen para imprevistos.

¿Cómo manejo las dependencias circulares? +

Dependencias circulares = ERROR en la planificación.

Ejemplo problemático: A depende de B, B depende de C, C depende de A

Soluciones:

  • Revisar lógica: ¿Realmente existe esa dependencia?
  • Dividir actividades: Separar en subtareas más específicas
  • Eliminar dependencias: Hacer actividades independientes
  • Replanificar secuencia: Cambiar el orden lógico

La aplicación detecta automáticamente estos errores y los reporta.

¿CPM vs PERT? ¿Cuál usar? +

Principales diferencias:

CPM (Critical Path Method):
  • Duraciones determinísticas (fijas)
  • Enfoque en costo y tiempo
  • Mejor para proyectos repetitivos
PERT (Program Evaluation Review Technique):
  • Duraciones probabilísticas (optimista/pesimista/más probable)
  • Enfoque en incertidumbre
  • Mejor para proyectos únicos/innovadores

Esta aplicación usa CPM por simplicidad educativa.

¿Qué tamaño de proyecto es recomendable? +

CPM es escalable, pero la complejidad crece exponencialmente.

Recomendaciones por tamaño:

  • 5-20 actividades: Ideal para aprender, fácil de visualizar
  • 20-50 actividades: Proyectos medianos, necesitas software
  • 50+ actividades: Proyectos grandes, herramientas profesionales

Consejo: Comienza con proyectos pequeños para dominar la técnica, luego escala.

Esta aplicación está optimizada para proyectos de aprendizaje (5-15 actividades).
